
Comandos área "cmd" do git
ls -> lista arquivos
cd -> move entre diretórios
pwd -> mostra diretório de trabalho
mkdir -> cria pasta no diretório
cat -> existe conteúdo do arquivo o que tem dentro de um arquivo
rm -rf.git -> exclui arquivo ou diretório de forma forçada


Comandos git:

git  init  -> transforma a pasta normal em "pasta especial" 
git status -> status dos arquivos que foram ou não versionados
git add    -> adicionado o que eu quero que va para o commit (ou o que eu quero que seja versionado)
git commit -m "Primeiros comandos bash" -> Cria e explica o commit (o que foi feito)

Git reset: Tira um arquivo do commit
Ex: git reset nomeArquivo.txt
Git checkout -b:  Cria uma brech secundaria.
ex: git checkout -b nomeBranchSegundaria
Git branch: Mostra todas as branch’s do projeto.
Git checkout main: Retorna pra breach main. (Não tem o -b, pq vc retorna. O -b serve para criar uma nova)
Git merge: Para trazer o que está em outra branch.
Ex: git merge branchSegundaria
Merge = misturar 
Git branch -D: Exclui a branch
Ex: git branch-d secundária

## Fluxo de trabalho

1. Criar diretório do projeto
2. Iniciar o git (git iniy.)
3. Edita ou cria arquivos
4. git status
5. git add <nome arquivo>
6. git status
7. git commit -m "mensagem"
8. git status
9. Volta para o passo 3.
